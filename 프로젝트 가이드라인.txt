1. 지능형 금융 상담&FSKU 평가 챗봇
- 컨텍스트에 따라 상담 모드 vs 평가 모드 자동 전환(Prompt Engineering로 구분 로직 설계)
상담 모드:
	- RAG기반 실시간 금융 정보 검색 및 답변
	- 복잡한 금융 용어 쉽게 설명
	- 다국어 지원
평가 모드:
	-FSKU평가지표 문항을 입력하면 객관식 정답 번호/주관식 답변 및 해설 생성
-하나의 vector DB에 상담용 자료와 FSKU평가 관련 문서를 임베딩해서 저장한 후,
사용자 질문에 따라 관련된 정보를 자동으로 검색해와서 답변 생성

2. 사용자 투자성향 분석
- 사용자의 투자 계획을 분석하여 과도한 위험 노출, 집중 투자, 시장 변동성 등을 감지하고
경고 메세지(위험도)
- 나이, 소득, 과거 투자 행동 등 수치 데이터로 투자 성향을 정확히 분류(안정, 보통, 위험)-머신러닝
- 분류 결과를 바탕으로 맞춤형 조언 생성(LLM)
- >개인화된 상세 설명 제공(ex:소득 대비 높은 위험 투자 비중과 기술주 선호 패턴을 보면 적극형 성향입니다. 
따라서 성장주 중심 포트폴리오를 추천하며...)

기술별 구현 방안

1. 임베딩 & Vector DB
임베딩:텍스트를 컴퓨터가 이해할 수 있는 숫자 벡터로 변환하는 기술
	(예: "주식 투자"와 "증권 투자"는 다른 단어지만 비슷한 의미로 벡터화됨)

실제 동작 과정:
- 1. 금융 문서들을 작은 덩어리(청크)로 나눔
- 2. 각 청크를 벡터로 변환 (예: 1536차원 숫자 배열)
- 3. Vector DB에 저장 (Chroma, Pinecone, Qdrant 중 선택)
- 4. 사용자가 "PER이 뭐야?"라고 물으면, 이 질문도 벡터로 변환
- 5. 저장된 벡터들 중 가장 유사한 것들을 빠르게 찾아서 반환

왜 필요한가:

수만 개의 금융 문서에서 관련 정보를 0.1초 만에 찾을 수 있음
키워드 검색과 달리 의미상 유사한 내용도 찾아줌

2. RAG
:Vector DB에서 관련 정보 검색하여 답변 생성

- 1. 데이터 소스 연동
- 2. 실시간 업데이터 파이프라인(매일 일정한 시간마다 새로운 공시자료 자동 수집 -> 변경된 내용만 vector DB에 업데이트)
- 3. context윈도우 최적화(사용자 질문에 가장 관련성 높은 5개 문서만 선별, 토큰 제한 내 최대한 많은 정보 포함)

3. 프롬프트 설계 예시
시스템 역할: 당신은 한국의 금융 전문가입니다.

사용자 레벨 감지:
	- "주식이 뭐예요?" → 초보자 모드 (쉬운 용어로 설명)
	- "PER 밸류에이션 분석해줘" → 전문가 모드 (전문 용어 사용)

FSKU 문항 감지:
	- "다음 중 옳은 것은?" → 객관식 모드 (정답 번호만 제시)
	- "ROE를 계산하시오" → 주관식 모드 (계산 과정 포함)

필수 포함 문구:
	- 모든 투자 조언 후: "투자에는 원금 손실 위험이 있습니다"
	- 법적 면책: "본 정보는 투자 권유가 아닌 참고용입니다"

응답 예시:
	-초보자: "PER은 주가를 주당순이익으로 나눈 값이에요. 쉽게 말해 그 회사 주식이 비싼지 싼지 알아보는 지표입니다."
	-전문가: "PER 15배는 현재 업종 평균 대비 저평가 구간으로, 밸류 투자 관점에서 매력적입니다."

4. Fine-Tuning
목적: 일반적인 AI 모델을 한국 금융 시장에 특화된 전문가 수준의 AI로 변환하는 것

방법:
	1. 데이터 준비(FSKU평가 문항, 한국 금융 지식 등)
	2. AI학습(AI에게 준비된 데이터를 반복적으로 보여주며 틀린 답변 수정하고 올바를 답변은 강화하는 방식으로 24시간 훈련 진행)
	3. 성능 검증(FSKU문제를 풀어보게 해서 정답률이 94%이상 나올 때까지 추가 학습 반복)

예시:
	질문:ROE계산해줘
	원래 답변: "자기자본이익률은 순이익을 자기자본으로 나눈 값입니다"
	->  "ROE = 순이익 ÷ 자기자본 × 100으로, 예를 들어 삼성전자 ROE가 10%라면 업종 평균 8% 대비 우수한 수준입니다"

5. 머신러닝
- 투자성향 분류 모델
예시:
	입력: 나이, 소득, 투자경험
	출력: 안정, 보통, 위험(30, 40, 30%)확률
	특성 중요도 분석

- FSKU앙상블 모델
	여러 개 모델의 예측을 결합하여 정확도 향상
	BERT + RoBERTa + GPT 앙상블로 객관식 정답률 95% 달성
	주관식은 생성형 모델 + 검증 모델 조합
